Scrapy基础

基础命令

  • 创建项目：scrapy startproject project_name
  • 创建爬虫：scrapy genspider spider_name xxx.com
  • 运行爬虫：scrapy crawl spider_name

  • 使用lxml解析方式
  • 支持CSS选择器，XPath选择器，正则表达式
  • 例子网址：https://www.bing.com/search?form=&q=python

正则

Spider的用法

  • 每个爬虫都是继承scrapy.Spider这个类
  • name ：唯一必须有
  • custom_settings ：本爬虫的配置信息，不同与settings中设置的会全局覆盖
  • from_crawler ：接受Crawler对象返回配置信息，设置到每一个爬虫上
  • start_requests ：一般会重写这个方法，但必须返回一个可以迭代的对象，默认call_back返回到parse
  • parse ：负责处理response
  • Request参数说明
  • 使用body对请求参数进行传递，使用meta对回调函数传参

Middleware的用法

下载中间件

  • 在Scheduler调度在把下载url交给下载器之前调用process_request可以对其请求头进行修改
    • 添加随机UA
    • 设置代理
  • 在下载器请求到response发送到spider之前在process_response可以手动处理响应结果

Item Pipeline 的用法

Item

  • 继承BaseItem，Meta中定义 table_name 和 唯一字段

Pipeline

  • 定义好了三个管道
  1. MysqlScrapyPipeline : 在存数据时，会先在数据库中查找unique_fields组成的条件是否可以取到记录，如果记录已经存在，则对数据库中的数据进行更新，如果不存在则向表中插入数据
  2. MongodbPipline : 将数据数据mongodb
  3. FieldsPipline : 将比较长的字段存入本地文件，并且将文件路径存入数据库

处理cookies

  • request
  • session
  • scrapy
    scrapy在settings中可以设置手动处理cookie
    • 默认关闭要手动开启
    • 开启后处理的cookies需要手动传入，注意是传给cookies，如果是加入headers中传入则scrapy会额外处理你的cookies。
    • 传递cookies格式

处理302跳转

  • 一般的302跳转都是重定向到下一层级的页面，所以处理这个只需要使用request，或session先请求重定向网址，不用接受结果，再请求其跳转的页面接受返回结果就行了，只需要注意每次提交的formdata，和每次提交的cookies。
